# Wage Compliance Analysis — Coastal Convenience Pty Ltd

A payroll compliance analysis in R, simulating the review of employer records against the General Retail Industry Award 2020 [MA000004] for a fictional three-store retail chain on the Central Coast of NSW.

## Violation Types

Two violation types are modelled, both affecting casual employees only and both consistent with genuine payroll misconfiguration rather than deliberate underpayment.

**Violation A — Penalty rate non-application (15 employees)**
The payroll system has no penalty rate rules configured. Every shift is paid at the ordinary rate regardless of day or time. Saturday, Sunday, public holiday, and evening loadings are never applied.

**Violation B — Below minimum rate (5 employees)**
The employer entered the pre-loading FT/PT rate instead of the correct all-in casual ordinary rate, not understanding that the 25% casual loading must be included in the recorded hourly rate. These employees are underpaid on every shift.

---

## Key Design Decisions

**Separation of employer records and award entitlements**
Entitlements are reconstructed from the award independently and never derived from what the employer paid. The analysis finds the gap between the two without any circularity.

**award_rate_id as the join key**
Each shift segment is pre-assigned an award rate ID during data preparation. The analysis script performs a simple join rather than conditional if/else logic — mirroring how real compliance systems work.

**Statistical detection via coefficient of variation**
The implied hourly rate (gross pay ÷ hours recorded) should vary across fortnights for correctly paid employees because penalty rates inflate gross pay on weekend and public holiday fortnights. Non-compliant employees show a flat implied rate. The coefficient of variation test independently identified all 20 underpaid employees using only payroll records and award rates.

## Setup
```r
# Required packages
install.packages(c("tidyverse", "lubridate"))

# Run scripts in order
source("01_build_dataset.R")
source("02_data_preparation.R")
source("03_compliance_analysis.R")
source("04_visualisation.R")
```

Scripts are fully self-contained. Running `01_build_dataset.R` generates all required input data for subsequent scripts.
